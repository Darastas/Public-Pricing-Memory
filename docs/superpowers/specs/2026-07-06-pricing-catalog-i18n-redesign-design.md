# Pricing Catalog, Bilingual UI, and Visual Polish Design

Date: 2026-07-06

## Goal

Add a curated public pricing directory alongside the existing crawler-based archive. The directory should make the app useful before crawls succeed by showing verified official pricing for well-known AI providers and major subscription platforms. The UI should support Chinese and English, fix visible icon/text overlap, and improve the frontend polish while preserving the product identity as a quiet pricing archive.

## User-Approved Approach

Use方案 B: add a separate curated pricing catalog data layer that is displayed next to, but not mixed into, crawler snapshots.

This keeps two data sources distinct:

- Crawler data: `Snapshot`, `PricingPlan`, `ChangeEvent`, and `CrawlJob` records generated by fetching public pages.
- Curated data: hand-maintained official price references with source URLs, regions, currencies, and last-checked dates.

Curated data must not be written into `Snapshot` or `PricingPlan` as if it came from a crawl.

## Source Policy

Pricing changes frequently. Implementation must verify pricing against official provider or platform pages at implementation time.

Allowed source types:

- Official pricing page.
- Official billing or help-center page.
- Official API price table.
- Official regional subscription page.

Disallowed as primary sources:

- Blog posts by third parties.
- Comparison sites.
- Search result snippets.
- Screenshots without official page context.
- Currency conversions not published by the vendor.

Every curated price item must store:

- `sourceUrl`
- `sourceLabel`
- `lastCheckedAt`
- `confidence`: `official` or `needs_review`

If official pages are ambiguous, unavailable, region-locked, or require login, include the product with source metadata and a `needs_review` price status instead of inventing values.

## Data Model

Add a static catalog module first, not a database migration.

Recommended files:

- `src/config/pricing-catalog.ts`
- `src/config/pricing-catalog.test.ts`
- `src/lib/catalog/filters.ts`
- `src/lib/catalog/filters.test.ts`

The catalog module should export typed data and helper selectors. This keeps the first version deployable without migration risk while leaving a path to migrate curated pricing into database tables later.

### Types

Use types shaped like this:

```ts
export type CatalogLocale = "en" | "zh";
export type CatalogRegion = "US" | "CN" | "EU" | "UK" | "JP" | "IN";
export type CatalogKind = "ai_api" | "ai_subscription" | "consumer_subscription" | "developer_subscription";
export type PriceStatus = "published" | "not_published" | "needs_review";

export type CatalogPrice = {
  id: string;
  kind: CatalogKind;
  label: { en: string; zh: string };
  region: CatalogRegion | "global";
  status: PriceStatus;
  amount: number | null;
  currency: "USD" | "CNY" | "EUR" | "GBP" | "JPY" | "INR" | null;
  billingPeriod: "month" | "year" | "one_time" | null;
  unit: string | null;
  rawText: string;
  sourceUrl: string;
  sourceLabel: string;
  lastCheckedAt: string;
  confidence: "official" | "needs_review";
};

export type CatalogProduct = {
  slug: string;
  name: string;
  category: CatalogKind;
  websiteUrl: string;
  pricingUrl: string;
  regions: CatalogRegion[];
  prices: CatalogPrice[];
};
```

`amount` stores a numeric value only when the source publishes a single clear numeric price. Token-based API prices can use `unit` and `rawText` to preserve exact vendor wording.

## UI Design

### Home Page

The home page remains the primary product surface and should not become a marketing page.

Add a pricing directory band between the hero/search area and recent crawler status:

- Bilingual title: "Pricing directory" / "价格目录"
- Compact filter row:
  - Category: all, AI API, AI subscriptions, consumer subscriptions, developer subscriptions
  - Region: global, US, CN, EU, UK, JP, IN
  - Search input
- Dense table or list with:
  - Product
  - Category
  - Region coverage
  - Representative price
  - Source status
  - Link to product detail

The existing recent changes and crawl status sections stay below the directory.

### Product Page

Add a "Official pricing references" / "官方价格参考" section near the current plans section.

Behavior:

- If crawler plans exist, show both crawler plans and official catalog references.
- If crawler plans are empty, the official catalog section still gives the user useful pricing data.
- Catalog prices should show source URL and last-checked date so users understand the data is curated.

### Header and Language Switcher

Add a compact language switcher in the header:

- `EN`
- `中文`

Use `?lang=en` and `?lang=zh` for the first implementation. Links should preserve the selected language where practical.

The root `<html lang>` value should reflect the active locale.

### Icon and Overlap Fixes

Current likely overlap risks:

- Header badges can crowd nav on medium widths.
- Badge text and lucide icons can compress inside small buttons.
- Table cells with many plan badges can overflow on mobile.

Fixes:

- Hide non-essential header badges earlier and keep the language switcher visible.
- Give icon buttons stable dimensions and prevent icon shrink with `shrink-0`.
- Let badge rows wrap with clear row gaps.
- Use responsive table wrappers with minimum widths only where tables need horizontal scroll.
- Avoid placing more than one compact badge cluster in the same narrow header row.

### Visual Direction

Keep the archive/productivity aesthetic, but make it more premium:

- Less heavy panel repetition.
- More structured section rhythm.
- Higher contrast typography.
- More deliberate status colors.
- Better table spacing and row grouping.
- A catalog table that feels like a market reference desk, not a generic admin table.

Do not introduce a large marketing hero, decorative blobs, or an AI chat-style layout.

## Bilingual Text Strategy

Do not add a heavy i18n framework in the first pass.

Recommended files:

- `src/lib/i18n.ts`
- `src/lib/i18n.test.ts`

Provide:

- `type Locale = "en" | "zh"`
- `getLocale(searchParams)`
- `dictionary`
- `withLocaleHref(href, locale)`

First pass text coverage:

- Header nav.
- Home page hero/search/metrics.
- Pricing directory filters and table labels.
- Recent changes and crawl status labels.
- Product page key headings and buttons.
- Empty states and database fallback messages.

Admin can remain mostly English in the first pass except shared header language controls, unless a touched string is easy to translate.

## Data Flow

Home page:

1. Parse locale and catalog filters from `searchParams`.
2. Load database-backed products, changes, and crawl jobs as today.
3. Load curated catalog products from static config.
4. Merge product identity by slug for display links.
5. Render catalog rows using selected locale, region, category, and query filters.

Product page:

1. Parse locale from `searchParams`.
2. Load product and crawler data as today.
3. Find matching catalog product by slug.
4. Render crawler plans and catalog references separately.

## Error Handling

- Database failure should not block catalog rendering because the catalog is static.
- Missing catalog source metadata should fail tests.
- Missing region price should render `Not published` / `未公开`.
- `needs_review` entries should render a neutral warning label, not a success label.

## Testing

Use TDD for implementation.

Required tests:

- Catalog data validation:
  - product slugs unique
  - price ids unique
  - source URLs parse as URLs
  - `published` entries have either a numeric amount or meaningful `rawText`
  - `not_published` entries do not use fake zero prices
  - every product has at least one source URL
- Catalog filters:
  - category filter
  - region filter
  - text search
  - locale-safe labels
- i18n helpers:
  - default locale
  - valid locale parsing
  - invalid locale fallback
  - preserving locale in links
- UI route tests or component-level checks where practical for the new selectors.

Rendered QA:

- Desktop home page: catalog visible, no header overlap, language switcher visible.
- Mobile home page: catalog filters and rows do not overlap.
- Product page with catalog data and no crawler plans still shows official pricing references.
- Language switch changes visible copy.

## Implementation Phases

1. Catalog types, validation tests, and initial official-source catalog data.
2. Seed product expansion tied to catalog slugs.
3. i18n helper and bilingual copy for touched surfaces.
4. Home page catalog section and responsive visual polish.
5. Product page official pricing references.
6. Browser QA and final verification.

## Acceptance Criteria

- Default seed includes the selected AI providers and subscription platforms.
- Home page shows a curated pricing directory without requiring crawls.
- Product page shows official pricing references separately from crawler-extracted plans.
- Catalog entries include source URL, last-checked date, status, region, currency, and price text.
- Chinese and English UI can be selected with a visible language switcher.
- Header/icons/badges do not overlap on desktop or mobile.
- Frontend feels more polished while preserving the archive identity.
- Tests cover catalog validation, filters, and i18n helpers.
- Final verification includes lint, typecheck, tests, build, and rendered QA.

## Out of Scope for First Pass

- Database migration for curated prices.
- User-submitted price corrections.
- Automatic official price page parsing into catalog data.
- Currency conversion for unpublished regional prices.
- Full admin translation.
- Subscription alerts, RSS feeds, or email notifications.
